# Debug Skill

协助排查问题时采用系统化方法：

## 调试流程

1. **复现** - 确认问题可稳定复现，记录复现步骤
2. **定位** - 缩小问题范围（前端/后端、模块、函数）
3. **假设** - 提出可能原因，设计验证实验
4. **验证** - 通过日志、断点、单元测试验证假设
5. **修复** - 实施最小化修复，添加回归测试

## 常用手段

- 二分法排除（注释一半代码，确认问题在哪侧）
- 添加日志输出关键变量和流程
- 对比正常与异常场景的差异
- 检查近期变更（git log、依赖更新）

## 输出建议

- 列出已尝试的步骤和结论
- 提出下一步排查方向
- 如能定位根因，说明原因并建议修复方案
