# Security Check Skill

进行安全审查时关注以下方面：

## 常见漏洞检查

1. **注入** - SQL、NoSQL、命令、LDAP 注入
2. **认证与授权** - 弱密码、会话固定、权限绕过
3. **敏感数据** - 硬编码密钥、日志泄露、传输未加密
4. **XSS** - 反射型、存储型、DOM 型
5. **CSRF** - 缺少 token 或同源验证
6. **依赖** - 已知漏洞的第三方库（检查 CVE）

## 审查要点

- 用户输入是否经过校验和转义
- 敏感操作是否有权限校验
- 错误信息是否泄露内部信息
- 配置是否包含生产凭据

## 输出格式

- 按严重程度分类（Critical / High / Medium / Low）
- 每项包含：问题描述、影响、修复建议、代码位置
