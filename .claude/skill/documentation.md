# Documentation Skill

编写技术文档时遵循以下标准：

## 文档类型与结构

### README
- 项目简介（一句话说明做什么）
- 快速开始（安装、运行最小示例）
- 主要功能/特性
- 目录结构说明
- 贡献指南链接

### API 文档
- 函数/类/接口的用途说明
- 参数说明（类型、含义、默认值）
- 返回值说明
- 使用示例
- 可能的异常/错误情况

### 设计文档
- 背景与问题陈述
- 方案对比与选择理由
- 详细设计（架构图、关键流程）
- 权衡与限制

## 写作原则

- **简洁** - 避免冗长，直击要点
- **准确** - 术语正确，逻辑严谨
- **示例** - 复杂概念配代码或图示
- **维护** - 代码变更时同步更新文档
