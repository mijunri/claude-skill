# Code Review Skill

当用户请求代码审查时，按以下维度进行系统化审查：

## 审查维度

1. **正确性** - 逻辑是否正确，是否有边界情况未处理
2. **安全性** - 是否存在漏洞（如 SQL 注入、XSS、敏感信息泄露）
3. **性能** - 是否存在 N+1 查询、内存泄漏、不必要的计算
4. **可读性** - 命名是否清晰，结构是否合理，注释是否恰当
5. **可维护性** - 是否遵循 SOLID 原则，是否有过度耦合
6. **测试** - 关键逻辑是否有测试覆盖

## 输出格式

- 按优先级列出发现的问题（Critical / High / Medium / Low / Suggestion）
- 每个问题附带：位置、描述、建议修改
- 总结优缺点和总体评价
