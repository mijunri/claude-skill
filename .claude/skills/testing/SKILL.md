# Testing Skill

编写测试时遵循以下实践：

## 测试金字塔

- **单元测试**（最多）- 快速、隔离、测试单一逻辑
- **集成测试**（适量）- 测试组件间协作
- **端到端测试**（少量）- 验证关键用户流程

## 单元测试原则

1. **独立性** - 测试之间不依赖执行顺序
2. **可重复** - 相同输入产生相同结果
3. **AAA 模式** - Arrange（准备）→ Act（执行）→ Assert（断言）
4. **命名清晰** - 描述测试场景，如 `should_return_error_when_user_not_found`

## 测试覆盖重点

- 核心业务逻辑
- 边界条件和异常路径
- 公共 API 和接口契约

## 避免

- 测试实现细节
- 过度 Mock 导致测试脆弱
- 重复的生产代码逻辑
