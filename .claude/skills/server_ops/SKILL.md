# 服务器操作 Skill

通过 SSH 操作远程服务器时遵循以下规范。

## 前置配置

- 在 **Cursor Cloud Agent** 中预先配置：
  - 服务器 IP 地址
  - 登录密码（或 SSH 私钥）
- 使用 `ssh` 命令或脚本连接服务器执行操作

## 连接方式

- 密码登录：`ssh user@<IP>`，按提示输入密码
- 私钥登录：`ssh -i /path/to/key user@<IP>`
- 避免在代码或脚本中硬编码 IP、密码，从环境变量或 Agent 配置读取

## 操作规范

- 执行命令前确认目标服务器无误（核对 IP、环境）
- 危险操作（rm、数据库变更、重启服务等）先 dry-run 或备份
- 记录关键操作的执行时间与结果，便于追溯
- 批量操作优先使用脚本，避免手工逐条执行

## 常见场景

- 查看日志：`tail -f /path/to/log`
- 重启服务：确认当前无重要任务后再执行
- 部署更新：先备份，再替换，最后验证
- 数据库操作：优先使用 `db_scripts/` 中的脚本，通过 SSH 在服务器执行
